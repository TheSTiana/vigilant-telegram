Sentiment analysis from social media

Introduction

A goal of sentiment analysis is to extract the opinion expressed by pieces of text, and it is a major area of applied natural language processing. In this project you will perform sentiment analysis on multiple different existing data sets and construct your own data set by scraping Twitter and produce sentiment labels for training supervised classification models. You are encouraged to try to both combine multiple data sets to train a single model, and to explore using transfer learning to increase classification performance.

The project will lead to both solid experience in data preparation for sentiment analysis based on the tools of fastai, and experience from a variety of different application domains.

Goals

  • Construct sentiment analysis models based on the NLP techniques of fastai.
  • Gain solid experience with the objectives, challenges, and pitfalls of this approach to sentiment analysis.
  • Optional: Dig into more recent developments in NLP, specifically the use of Transformers, using the HuggingFace library.

Methods and materials

A major part of the project is to investigate a variety of data sets, as this is the best way to gain solid, generalizable experience. Here are some existing data sets to get you started (you should look for additional data sets yourself):

  • Amazon Review Data. See also URLs.AMAZON_REVIEWS and URLs.AMAZON_REVIEWS_POLARITY in fastai
  • Twitter US Airline Sentiment
  • Multi-Domain Sentiment Dataset
  • OpinRank Review Dataset

You are encouraged to construct your own data set by scraping one or more relevant internet sources. The Python libraries Beautiful Soup and Scrapy are worth investigating. Twitter is a relatively easy source for data, perhaps combined with efficient techniques for data labeling, for example Snorkel.
